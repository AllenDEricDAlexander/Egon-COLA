# Student Management Organization

This project is generated by `egon-cola-archetype-web` as one independent, organization-only Project. It publishes its own Facade artifact and consumes the Evaluation Facade only through an Infrastructure adapter.

${symbol_pound}${symbol_pound} Modules

```text
student-management-organization-common
student-management-organization-facade
student-management-organization-domain
student-management-organization-application
student-management-organization-infrastructure
student-management-organization-adapter
student-management-organization-starter
```

${symbol_pound}${symbol_pound} Dependency Direction

```text
common         -> no generated module
facade         -> no generated module
domain         -> common
application    -> domain
infrastructure -> domain
adapter        -> application, facade
starter        -> adapter, infrastructure
```

Infrastructure implements Domain-owned ports. Adapter never reaches Infrastructure directly, and Starter contains assembly configuration only.

${symbol_pound}${symbol_pound} Domains

The complete `user` vertical creates and queries users, assigns roles, grants permissions, caches user reads, and publishes committed changes.

The complete `teaching` vertical creates and queries grades and school classes, caches reads, and publishes committed changes. User-to-class assignment is one Application transaction using both Domain repositories; `school_class_users` is the authoritative membership relationship.

${symbol_pound}${symbol_pound} Integration Ownership

- Adapter owns HTTP `/api/v1/**`, GraphQL `/graphql`, inbound RabbitMQ commands, Dubbo Facade export, request validation, filters, and protocol conversion.
- Infrastructure owns JPA, Flyway, Redis adapters, outbound RabbitMQ events, the Evaluation Facade anti-corruption adapter, local fallback adapters, and Application-method logging AOP.
- Starter owns OpenAPI assembly, runtime profiles, Actuator, Prometheus, Jackson, async execution, and configuration decryption.
- Facade is the dependency-safe contract for external consumers and contains no implementation.

RabbitMQ command retries use three total attempts with bounded backoff and dead-letter queues. Domain events publish after commit. This example reports exhausted event publication but does not claim transactional-outbox delivery.

${symbol_pound}${symbol_pound} Runtime Profiles

`local` and `test` use H2 in PostgreSQL compatibility mode, in-memory cache/idempotency adapters, a local event publisher, a deterministic Evaluation query stub, disabled RabbitMQ and Nacos connections, and Dubbo `injvm` with no registry.

`dev` and `prod` use the Dubbo Evaluation Facade client with a 3000 ms timeout, zero retries, and startup reference checks. Facade groups and version use `EVALUATION_*_FACADE_GROUP` and `EVALUATION_FACADE_SERVICE_VERSION`; other datasource, Redis, RabbitMQ, Nacos, cache, and idempotency settings remain environment-backed.

${symbol_pound}${symbol_pound} Error Contract

HTTP failures use a stable body containing `code`, `message`, `traceId`, `timestamp`, and `fieldErrors`. Status semantics are `400` validation, `403` forbidden, `404` not found, `409` conflict, `422` Domain rejection, `503` required dependency unavailable, and `500` unexpected failure. GraphQL exposes the same fields through error extensions; Facade failures carry the same code and trace ID.

${symbol_pound}${symbol_pound} Commands

Verify all modules:

```bash
bash ./mvnw -V --no-transfer-progress clean verify
```

Package the layered executable jar:

```bash
bash ./mvnw -V --no-transfer-progress -DskipTests package
```

Encrypt a configuration value:

```bash
CONFIG_DECRYPT_KEY=base64-encoded-32-byte-key \
  bash ./mvnw -pl ${rootArtifactId}-starter \
  -Dexec.mainClass=${package}.starter.config.encryption.ConfigCipherCli \
  -Dexec.args='encrypt plaintext-value' exec:java
```

Docker build:

```bash
docker build -t ${rootArtifactId}:local .
```

Optional local run:

```bash
SPRING_PROFILES_ACTIVE=local bash ./mvnw -pl ${rootArtifactId}-starter spring-boot:run
```

Sensitive values belong in environment variables, mounted files, `config/application-secrets.yml`, or `configtree:/run/secrets/`. Do not commit credentials or decryption keys.
